# MIAS Overview

晓东博士关于机器人算法开发的几点想法和规划:

  1. # 如何打造一个有价值的平台？

    ​ 旷通软件平台的价值在于提供更优的算法模块。对于基础架构来说，不同平台都大差不差，但是如果在很多子模块的功能上，我们提供的模块都比其它渠道的模块功能更强、环境适应性更好，用户能得到更稳定可靠的结果，自然就会吸引客户聚集在旷通的软件平台上。 ​ 例如: 光流算法、视差图算法、IMU航迹算法、图像与IMU紧耦合算法、轮子里程算法（打滑、转弯等环节）、语义地图等等。每个子模块的优化没有弯道可以超车，都需要通过大量基础性的测试进行归纳研究，细分成不同环境条件，并逐一优化参数。这些累积起来会构成旷通软件平台的技术门槛。

  2. # 语义地图是智能化平台的核心，对语义地图的顶层设计要优于编程技巧。

    ​ 机器人的轨迹要能够和地图结合起来，机器人二次进入环境后，需要能够通过查询语义地图，确定自身精确位置，这样的地图才真正有效。 ​ A: 语义地图应该是模拟一个成年人判断路径的思考过程。 ​ B: 语义的定义需要覆盖人、车、桌椅等独立的小个体，也需覆盖草坪、树丛、水面等连续的大片物体。 ​ C: 机器人生成的局部语义（视线范围）和整体语义地图做匹配时，要有足够的模糊判断能力。 ​ D: 要允许机器人临时迷失方向，带有一定猜测去探路

    在语义地图中定位、搜索路径的过程，就是机器人“大脑”思考的过程。我们要敢于突破现有语义地图的定义，做一些全新的尝试。

  3. # 对开发顺序的规划

    ​ A: 基于GPS数据建图（本质也是一种语义地图），搭建基础软件架构，实现最简条件下机器人出发-巡航-返回的完整过程。 ​ B: 使用CEVA模块，以”IMU+轮速+光流+GPS”为基础数据实现出发-巡航-返回的完整过程。 ​ C: 参考VINS的”光流/深度图+IMU”方法，生成机器人轨迹，并与A、B中的轨迹做综合判断。 ​ D: 定义语义地图规则，包括单一物体的语义定义、多个物体之间的语义联系、语义匹配时的模糊判断等。 ​ E: 使用mask rcnn做物体识别、大片区域（草地、道路、水面）的边界识别，并使用双目立体测距，形成语义地图核心元素。

    ​ 以上几点想法供大家探讨，希望大家的工作更聚焦，集中力量将基础性的储备工作完成好，咱们团队整体的研发速度才能提起来。
